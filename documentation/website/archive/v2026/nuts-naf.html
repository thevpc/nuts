Maven Integration

One dependency. Full runtime. Just add this to your pom.xml and NAF takes care of the rest.

<dependency>
  <groupId>net.thevpc.nuts</groupId>
  <artifactId>nuts</artifactId>
  <version>0.8.9</version>
</dependency>

<!-- Add thevpc repo -->
<repository>
  <id>thevpc</id>
  <url>https://maven.thevpc.net</url>
</repository>
// One line to start
Nuts.require();
NOut.println("Hello Nuts!");
Rich Messaging

NMsg — Structured, Styled Output

NMsg creates structured messages with dynamic placeholders, automatic type coloring (booleans, numbers, paths, dates), custom styles, and nested composition. Supports C-style, Java-style, and variable-substitution placeholder formats.

Multiple Placeholder Formats

C-style, SLF4J-style, or named variable substitution — choose what fits your style.

NMsg.ofC("Hello %s, you have %d notifications", "Alice", 5);
NMsg.ofJ("Downloading {} from {}", "report.pdf", "server-01");
NMsg.ofV("User $user on ${app}", Map.of("user", "Alice"));

Text Rendering Formats

Plain, styled, code blocks, or full NTF markup with bold, italic, and colors.

NMsg.ofPlain("Plain output");
NMsg.ofStyled("Error: file not found", NTextStyle.error());
NMsg.ofCode("java", "System.out.println(\"Hello\")");
NMsg.ofNtf("##bold## ##:/:italic## ```code```");
Filesystem

NPath — Unified Local & Remote Paths

Work with local files, HTTP, and SSH using the same API. Access standard system directories cross-platform without writing OS-specific code.

// Local and remote paths — same API
String content = NPath.of("http://myserver/myfile.txt").readString();
String[] lines = NPath.of("ssh://myserver/myfile.txt").lines().toArray(String[]::new);

// Standard directories
NPath.ofUserHome();                              // ~/
NPath.ofUserStore(NStoreType.CONFIG);           // ~/.config/nuts on Linux
NPath.ofSystemStore(NStoreType.BIN);            // /opt/nuts on Linux
CLI

NCmdLine — Clean Argument Parsing

Parse flags, options, and non-option arguments with full type handling. Build sophisticated CLI tools without the parsing boilerplate.

NCmdLine cmdLine = NApp.of().getCmdLine();
NRef<Boolean> boolOpt = NRef.of(false);
NRef<String> strOpt  = NRef.ofNull();
while (cmdLine.hasNext()) {
    cmdLine.matcher()
        .with("-o", "--option").matchFlag(v  -> boolOpt.set(v.booleanValue()))
        .with("-n", "--name").matchEntry(v   -> strOpt.set(v.stringValue()))
        .withNonOption().matchAny(v          -> strOpt.set(v.image()))
        .requireDefaults();
}
Logging

NLog — Scoped, Context-Aware Logging

Beyond MDC: nested logging scopes, dynamic placeholders, message prefixes, and custom log handlers — all without touching global configuration. Integrates with JUL and SLF4J.

Structured Log Entries

Attach intent, exceptions, and duration to every log message.

NLog.of("myapp").info(
  NMsg.ofV("Task %s completed", "Upload")
    .withIntent(NMsgIntent.CONFIG)
    .withThrowable(exception)
    .withDurationMs(123)
);

Scoped Logging

Logs automatically inherit the correct context without explicit parameter passing.

NLog.runInScope(
  NLogScope.of()
    .withMessagePrefix(NMsg.ofC("[App]"))
    .withPlaceholder("user", "Adam"),
  () -> OtherClass.doThis()
);
Resiliency

Resilience Patterns Out of the Box

NAF ships production-grade resilience primitives: rate limiting, circuit breakers, retry logic, saga orchestration, caching, and locking — all cohesive and composable.

NRateLimitedValueNCircuitBreakerCall NRetryCallNSagaCall NCachedValueNStableValue NLockNBulkheadCall 
// Rate limiting: 10 calls per 2 minutes, sliding window
NRateLimitedValue limiter = NRateLimitedValue.ofBuilder("api")
    .withLimit("calls", 10).per(Duration.ofMinutes(2))
    .withStrategy(NRateLimitDefaultStrategy.SLIDING_WINDOW)
    .build();
limiter.claimAndRun(() -> NOut.println("Action executed"));

// Process-safe file lock
NLock.ofPath(NPath.of("/path/to/resource.txt"))
    .runWith(() -> NOut.println("Critical section"));

NElementFormat.ofPlainXml(doc)
