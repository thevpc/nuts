Nuts Application Framework

v0.8.9.0Java 8+ Spring-compatibleSLF4J-compatible

NAF (Nuts Application Framework) is a lightweight, modular framework built on top of the Nuts runtime. It provides a unified foundation for configuration, logging, dependency injection, modularity, and extensibility — all while remaining Java 8 compatible.

NAF is non-invasive by design: it integrates with Spring, Spring Boot, SLF4J, and other frameworks without requiring you to rewrite or restructure your application.

Installation

Add NAF to your project with a single Maven dependency:

XML — pom.xml
<dependency>
  <groupId>net.thevpc.nuts</groupId>
  <artifactId>nuts</artifactId>
  <version>0.8.9</version>
</dependency>

<repositories>
  <repository>
    <id>thevpc</id>
    <url>https://maven.thevpc.net</url>
  </repository>
</repositories>

Quick Start

One line initializes the full NAF runtime with workspace, dependency management, and I/O tools ready:

Java
Nuts.require();                         // Initialize NAF runtime
NOut.println("Hello Nuts!");              // Styled console output

// Or open a custom workspace
NWorkspace ws = Nuts.openWorkspace("--workspace=/path/to/ws");
ws.runWith(() -> {
    NSession.of().copy()
        .setOutputFormat(NContentType.JSON)
        .runWith(() -> NOut.println(Map.of("status", "ok")));
});
TipUse Nuts.require() when you need a fast in-memory workspace without any filesystem setup — perfect for embedded usage and testing.

NMsg — Structured Messages

NMsg is NAF's message abstraction. It supports multiple placeholder formats, automatic type-aware coloring, and message composition.

Placeholder Formats

Java
// C-style printf format
NMsg.ofC("Hello %s, you have %d notifications", "Alice", 5);

// SLF4J-style {}
NMsg.ofJ("Downloading {} from {}", "report.pdf", "server-01");

// Named variable substitution
NMsg.ofV("User $user on ${app}", Map.of("user", "Alice", "app", "NAF"));

// Dynamic supplier
NMsg.ofV("Threshold=$th, Date=$date", name -> switch (name) {
    case "th"   -> 0.85;
    case "date" -> LocalDate.now();
    default    -> null;
});

NPath — Unified Paths

NPath provides a unified API for local files, HTTP, and SSH paths. Access standard system directories without OS-specific code.

Java
// Remote paths work just like local paths
String content = NPath.of("http://server/file.txt").readString();
String[] lines = NPath.of("ssh://server/file.txt").lines().toArray(String[]::new);

// Standard directories — cross-platform
NPath.ofUserHome();
NPath.ofUserStore(NStoreType.CONFIG);   // ~/.config/nuts on Linux
NPath.ofSystemStore(NStoreType.BIN);    // /opt/nuts on Linux
SSH NoteSSH paths require the nlib-ssh extension. For remote Windows hosts, use ssh://host?use=bin or ?use=portable to select the execution mode.

NLog — Scoped Logging

NAF extends standard MDC with nested logging scopes, dynamic placeholders, and custom log handlers. Logs automatically inherit the correct context.

Java
// Standard structured log entry
NLog.of("myapp").info(
    NMsg.ofC("Task %s done with status %s", "Upload",
        NText.ofStyled("OK", NTextStyle.primary1()))
    .withIntent(NMsgIntent.CONFIG)
    .withDurationMs(123)
);

// Scoped logging — context inherited automatically
NLog.runInScope(
    NLogScope.of()
        .withMessagePrefix(NMsg.ofC("[MyApp]"))
        .withPlaceholder("user", "Adam")
        .withLog(msg -> NOut.println(NMsg.ofC("[SCOPED] %s", msg))),
    () -> OtherClass.doThis()
);
