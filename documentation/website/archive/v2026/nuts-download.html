Run Nuts in a container

Perfect for ephemeral testing, CI/CD pipelines, or isolated builds. No local install required.

One-liner: ephemeral shell with Nuts
$ docker run -it --rm openjdk:8 bash -c "$(curl -sSL https://thevpc.net/nuts/bootstrap-container-latest.sh)"
Then inside the container:
nuts -y <your-app> [args]

Example (OpenAPI → PDF):
nuts -P=%s -ZyS net.thevpc.nuts.toolbox:noapi#https://maven.thevpc.net/net/thevpc/nuts/nuts-app/0.8.9/nuts-app-0.8.9.jar myrest-apis.json
Minimal Dockerfile (bootstrap script)
FROM openjdk:8
RUN curl -sSL https://thevpc.net/nuts/bootstrap-container-latest.sh | bash
RUN nuts -Zy install <your-app>
CMD ["nuts", "-y", "<your-app>"]
Alternative: bundle JAR directly
FROM openjdk:8
RUN wget https://maven.thevpc.net/net/thevpc/nuts/nuts-app/0.8.9/nuts-app-0.8.9.jar -qO nuts.jar
RUN java -jar nuts.jar -Zy install net.thevpc.nuts.toolbox:noapi
CMD ["java", "-jar", "nuts.jar", "-y", "net.thevpc.nuts.toolbox:noapi"]
